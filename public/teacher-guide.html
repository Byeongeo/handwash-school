준비물 내 앱 만들기 샘플 수집 학생 사용 문제 해결

초보 교사용 순서 안내

이 문서대로 따라 하면 내 Google Sheets와 내 Vercel 앱을 소유하고 사용할 수 있습니다

이 앱은 손씻기 동작을 휴대폰·패드 카메라로 수집하고, 학생이 손을 씻을 때 부족한 단계를 음성으로 알려주며, 완료 기록을 Google Sheets에 저장합니다.

/collect

교사가 손씻기 동작 샘플을 모으는 화면입니다. 학생에게 공개하지 않는 것이 좋습니다.

/student

학생용 키오스크 화면입니다. 발판·터치·QR로 시작해 단계별 안내를 듣고 점수를 받습니다. (/wash는 교사 점검용)

/teacher

샘플 개수와 최근 학생 기록을 확인하는 교사용 화면입니다.

Google Sheets

학생 명단, 손동작 샘플, 학생 완료 기록이 저장되는 곳입니다.

먼저 확인

1. 준비물

  • Google 계정: Google Sheets 사본과 Apps Script 배포에 필요합니다.
  • GitHub 계정: Vercel Deploy Button으로 앱 저장소를 복사할 때 필요합니다.
  • Vercel 계정: GitHub 계정으로 로그인하면 됩니다.
  • 카메라가 있는 휴대폰 또는 패드: 실제 수돗가에서 샘플을 모으고 학생용 앱을 실행합니다.
  • 인터넷 연결: MediaPipe 모델과 앱을 불러올 때 필요합니다.
  • 투명 지퍼백 또는 방수 케이스: 물가에서 기기를 보호합니다.
중요: 손씻기 판정은 앱 안의 브라우저에서 바로 처리합니다. Google Sheets는 샘플 묶음 저장과 학생 완료 기록 저장에만 사용합니다.

Google이 "확인되지 않은 앱"이라고 표시할 수 있습니다. 연수용으로 본인이 만든 시트에 접근하는 코드입니다. 고급 → 프로젝트 이름으로 이동 → 허용 순서로 진행합니다.

항목 선택값 이유
실행 계정 나 내 Google Sheets 사본에 기록하기 위해서입니다.

배포 후 나오는 /exec로 끝나는 웹앱 URL을 복사해 둡니다. 다음 단계에서 APPS_SCRIPT_URL에 붙여넣습니다.

4. 휴대폰·패드로 샘플 수집하기

수집 전에 기기를 놓는 방법

위치 실제 학생이 손을 씻을 수돗가에 휴대폰이나 패드를 놓습니다. 손이 화면 중앙에 들어와야 합니다.
각도 손을 위에서 약간 내려다보는 각도가 좋습니다. 정면 셀카 각도는 손가락 구분이 어렵습니다.
노트북을 물가에 들고 갈 필요가 없습니다. 휴대폰이나 패드로 /collect를 열어 수집하면 됩니다. 실전에서 방수팩을 씌워 쓸 기기라면 방수팩을 씌운 상태로 수집해야 판정 조건이 같아집니다.
9

/collect 열기

휴대폰 또는 패드 브라우저에서 내앱주소/collect를 엽니다. 처음 열면 카메라 권한을 묻습니다. 반드시 허용을 누릅니다.

10

상단 왼쪽의 카메라 시작 누르기

화면 왼쪽 큰 카메라 영역에 손이 보이는지 확인합니다. 아래쪽의 손 감지 숫자가 1 또는 2로 나오면 손을 인식하고 있는 상태입니다.

기본은 화면(액정) 쪽 전면 카메라이고 거울처럼 표시됩니다. 바깥쪽 카메라가 나오면 카메라 전환 버튼을 누르세요. 이 선택은 같은 기기·같은 브라우저의 학생 화면(/student)에도 기억됩니다. 샘플 수집과 학생 실행은 반드시 같은 카메라 방향을 사용합니다.
11

수집 설정 입력하기 (한 번만 · 자동 저장)

입력칸 예시 설명
샘플 세트 이름 3학년_보건실 나중에 Google Sheets config 탭의 activeSampleSet에 이 이름을 넣습니다.
기기/장소 메모 보건실 패드 어떤 기기와 위치에서 수집했는지 적습니다.
현재 라벨 1 손바닥 마주대기 지금 수집할 손동작. 카메라 위 툴바의 드롭다운에서 고릅니다.
12

카메라 위 툴바에서 바로 수집하기

카메라 화면 위 툴바에 현재 라벨 드롭다운과 수집 시작 / 수집 완료 버튼이 있어, 휴대폰에서 화면을 오르내리지 않고 수집할 수 있습니다. 옆에 "이 라벨 n개 · 전체 n개"로 저장 개수가 실시간 표시됩니다.

라벨을 고르고 수집 시작을 누른 뒤 손동작을 조금씩 움직이며 유지하면 약 0.3초 간격으로 자동 저장되고, 100개가 되면 음성 안내와 함께 저절로 완료됩니다(약 30초). 버튼에 진행 개수(n/100)가 표시되며, 중간에 멈추려면 버튼을 다시 누릅니다. 같은 라벨을 더 모으려면 다시 수집 시작을 누르면 100개 단위로 추가됩니다. 오른쪽 패널의 1개 저장은 현재 프레임 하나만 저장합니다.
샘플 세트 이름·기기 메모·현재 라벨은 기기에 자동 저장되어, 앱을 다시 열어도 그대로 유지됩니다.
13

7개 라벨 모두 수집하기

아래 7개 라벨을 모두 모읍니다. 기타/대기는 꼭 필요합니다.

라벨 수집할 장면 권장 개수
손바닥손바닥과 손바닥을 마주 대고 문지름50개 이상
손등손등을 반대 손바닥으로 문지름50개 이상
깍지손가락 사이를 끼고 문지름50개 이상
손가락손가락을 마주 잡고 문지름50개 이상
엄지엄지를 돌려 문지름50개 이상
손톱/손목손끝이나 손목 부분을 문지름50개 이상
기타/대기손 없음, 물만 틀기, 손을 가져오는 중50개 이상
14

묶음 업로드 누르기

오른쪽 아래의 수집 코드 칸에 Vercel 환경변수 TRAINER_CODE로 정한 값을 입력하고, 묶음 업로드를 누릅니다.

업로드 전에는 샘플이 브라우저 안에만 임시 저장됩니다. 수집이 끝나면 반드시 묶음 업로드를 누릅니다. 샘플이 많으면 400개씩 자동으로 나눠 올라가며, 끝나면 음성 안내와 함께 임시 샘플 개수가 0이 됩니다.
15

사용할 샘플 세트 지정하기

Google Sheets로 돌아가 config 탭을 엽니다. activeSampleSet 값에 방금 수집한 샘플 세트 이름을 정확히 입력합니다.

예시 activeSampleSet 값: 3학년_보건실

학생 사용

5. 학생이 손씻기 앱 사용하기

16

/student 열기 (학생용 키오스크)

수돗가에 고정한 공용 패드나 휴대폰(투명 방수팩)에서 내앱주소/student를 엽니다. 상단 메뉴가 없는 학생 전용 전체 화면이라 학생이 다른 화면으로 이동할 수 없습니다.

/wash는 교사 점검·연수 실습용으로 남아 있습니다. 수동 학생 ID 입력과 상세 패널이 필요할 때 사용합니다.
17

카메라 켜는 방법 고르기

기본: 발판 또는 화면 터치 학생이 발판(블루투스 풋스위치)이나 화면을 누르면 카메라가 켜집니다. 처음 한 번 카메라 권한을 물으면 허용을 누릅니다.
무접촉: 항상 대기 모드 Google Sheets config 탭의 alwaysOn을 Y로 바꾸면, 교사가 아침에 화면을 한 번 눌러 켠 뒤 하루 종일 QR 대기 상태가 유지됩니다. 학생은 아무것도 만지지 않습니다.
항상 대기 모드는 충전기를 상시 연결하고, 직사광선을 피해 거치하세요.
18

QR 명찰로 학생 인식

카메라가 켜지면 학생이 QR 명찰을 화면에 보여줍니다. 이름을 부르며 자동으로 세션이 시작됩니다. QR은 Google Sheets의 student list 탭에서 자동 생성되므로 인쇄해 명찰로 사용합니다.

QR이 없을 때는 교사 점검용 /wash에서 수동 학생 ID 칸에 3-2-15 홍길동처럼 입력해 시작할 수 있습니다.
19

학생이 손씻기 6단계 수행

오른쪽 진행 상황 영역에서 손바닥, 손등, 깍지, 손가락, 엄지, 손톱/손목 진행 막대가 올라갑니다. 부족한 단계는 음성으로 안내됩니다.

예: "엄지 단계를 더 해 주세요.", "손바닥 단계 완료"
손씻기 도중 자리를 떠나 손이 idleTimeoutSec(기본 20초) 동안 보이지 않으면 세션이 자동 취소되고 기록이 남지 않습니다.
20

완료 기록 확인

6단계를 모두 목표 시간 이상 수행하면 점수가 부여되고 Google Sheets의 records 탭에 기록됩니다. 앱의 /teacher 화면에서도 최근 기록을 확인할 수 있습니다.

관리

6. 교사용 확인 화면과 시트 관리

/teacher 화면

샘플 라벨별 개수와 최근 학생 기록을 빠르게 확인합니다.

  • 샘플 개수가 한 라벨만 지나치게 적으면 /collect에서 보강합니다.
  • 최근 기록이 보이지 않으면 Apps Script URL과 Vercel 환경변수를 확인합니다.

Google Sheets 탭

  • student list: 학번, 이름, QR
  • samples: 손동작 좌표 샘플
  • records: 학생 완료 기록
  • config: 점수, 목표 초, 샘플 세트 설정

config 탭에서 자주 바꾸는 값

키 예시 값 설명
pointsPerCompletion 10 완료 시 받을 점수입니다.
requiredSeconds 3 각 단계가 인정되기 위해 필요한 시간입니다.
confidenceThreshold 65 판정 신뢰도 기준입니다. 너무 엄격하면 잘 안 올라가고, 너무 낮으면 오판이 늘어납니다.
activeSampleSet 3학년_보건실 학생용 앱에서 불러올 샘플 세트 이름입니다.
idleTimeoutSec 20 손이 안 보인 채 이 시간이 지나면 세션을 자동 취소합니다(기록 없음).
alwaysOn Y 또는 N Y면 학생 키오스크가 하루 종일 카메라 QR 대기 상태를 유지합니다(무접촉 시작).
allowUnregistered Y 또는 N 명단에 없는 학번을 허용할지 정합니다.

문제 해결

7. 자주 막히는 부분

카메라가 안 켜질 때

  • 브라우저 주소가 https://인지 확인합니다. Vercel 주소는 HTTPS입니다.
  • 브라우저 주소창 왼쪽의 자물쇠 또는 카메라 아이콘에서 권한을 허용합니다.
  • 다른 앱이 카메라를 사용 중이면 닫습니다.
  • iPhone/iPad에서는 Safari 또는 Chrome 최신 버전을 사용합니다.

샘플 업로드가 안 될 때

  • TRAINER_CODE와 수집 페이지의 수집 코드가 같은지 확인합니다(Vercel에 설정하지 않았다면 빈 칸이면 됩니다).
  • Vercel 환경변수 APPS_SCRIPT_URL이 /exec로 끝나는지 확인합니다.
  • Apps Script 배포 권한이 모든 사용자인지 확인합니다.
  • 샘플이 많아도 앱이 400개씩 자동으로 나눠 올립니다. 중간에 끊기면 저장 안 된 나머지만 남으니 다시 눌러 이어서 올립니다.

학생 기록이 시트에 안 남을 때

  • Google Sheets의 records 탭이 있는지 확인합니다.
  • Apps Script에서 setup을 실행했는지 확인합니다.
  • Vercel 배포 후 환경변수를 바꿨다면 다시 배포해야 할 수 있습니다.

동작 판정이 자꾸 틀릴 때

  • 실제 사용할 위치에서 샘플을 다시 모읍니다.
  • 틀리는 라벨을 50개 이상 추가합니다.
  • 기타/대기 샘플을 충분히 모읍니다.
  • 조명이 너무 어둡거나 역광이면 위치를 조정합니다.

운영 주의

8. 개인정보와 수업 운영

  • 앱은 사진이나 영상을 Google Sheets에 저장하지 않습니다. 손 관절 좌표 숫자와 완료 기록을 저장합니다.
  • 학생 이름과 학번은 학교 개인정보 처리 기준에 맞게 최소한으로 사용합니다.
  • 공용 기기는 물이 튀지 않는 위치에 고정하고, 투명 방수 케이스를 사용합니다.
  • 점수는 위생 교육 보조 자료로 사용하고, 의학적 판단이나 공식 위생 검증으로 사용하지 않습니다.
